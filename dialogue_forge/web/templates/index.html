What is DLG?

DLG is a simple, human-readable format for writing branching dialogue for RPGs and adventure games. Use @talk to define when the player can talk to NPCs, and @end to return control to gameplay.

NPC Conversations

The core RPG pattern: @talk defines when dialogue triggers, @end exits back to gameplay.

[characters]
hero: Player
officer: Fire Nation Officer
recruit: Recruit

# ==========================================
# OFFICER - Multiple conversations based on state
# ==========================================

[officer_intro]
@talk:officer {!talked_to_officer}
officer: "On your feet, recruit!"
-> yes_sir: "Yes, sir!"
-> confused: "Where are we?"

[yes_sir]
*set talked_to_officer = true
officer: "Your gear's in that locker. Suit up!"
@end

[confused]
*set talked_to_officer = true
officer: "Wake up! Get your gear from the locker!"
@end

[officer_ready]
@talk:officer {talked_to_officer && has_item:uniform}
officer: "About time. Get to the deck."
-> deck

[officer_waiting]
@talk:officer {talked_to_officer && !has_item:uniform}
officer: "Still no gear? The locker, recruit!"
@end

# ==========================================
# RECRUIT - Simple conversation
# ==========================================

[talk_recruit]
@talk:recruit
recruit: "Nervous about the mission?"
@end

üí° @talk triggers are evaluated in order - the last matching one wins. Put your most specific conditions at the bottom!

File Structure

Every DLG file has three sections: characters, state (optional), and nodes.

[characters]
hero: Player Name
npc: Village Elder

[state]
*set talked_before = false
*set gold = 100

[start]
@talk:npc
npc: "Welcome, traveler!"
-> greet: "Hello!"
-> leave: "Goodbye."

[greet]
*set talked_before = true
npc: "Nice to meet you!"
@end

[leave]
npc: "Safe travels."
@end

State Initialization

The [state] section lets you define initial game state for testing and documentation:

[state]
# Set variables
*set has_key = false
*set reputation = 0
*set player_name = Korra

# Give items
*give_item torch
*give_item map

# Add companions
*add_companion peng

üí° The [state] section is executed before any dialogue starts. Use it to set up the scene's required state for testing!

State carries between scenes in the game. The [state] section documents what state this dialogue expects and provides defaults for testing in the editor.

Dialogue Lines

Each line has a speaker and quoted text:

npc: "What brings you here?"
hero: "I'm looking for adventure!"
narrator: "The wind howls outside."

üí° narrator is special ‚Äî it shows as italic scene description.

Multi-line Dialogue

For long dialogue, split across multiple lines by leaving the quote open:

elder: "Welcome to our village, traveler.
    We have been expecting you for some
    time now. Please, make yourself
    at home."

The lines are joined with spaces. You can also add a condition at the end:

elder: "I see you've brought the sacred
    artifact. This changes everything." {has_item:artifact}

üí° Indent continuation lines for readability. Empty lines and comments within multi-line text are skipped.

Tags (Optional)

Add optional metadata tags in square brackets for game integration:

# Emotion tags for character portraits
peng: "I finally found you!" [happy]
hero: "Is that really you?" [surprised]

# Multiple tags
peng: "It was so hard..." [sad, tired]

# Tags with conditions
peng: "Thank you!" [grateful, tearful] {saved_peng}

Common uses:

  • Emotions: [happy], [sad], [angry]
  • Portrait states: [mask_on], [wounded]
  • Scene hints: [whisper], [shouting]

üí° Tags are optional metadata ‚Äî your game decides how to use them (e.g., switching character portraits based on emotion).

Choices & Transitions

Use -> to branch. A colon means player choice, no colon means automatic GOTO:

# PLAYER CHOICES (colon + text)
-> next_node: "What the player says"
-> fight: "[Attack the enemy]"

# AUTOMATIC TRANSITIONS (no colon = GOTO)
-> another_node          # unconditional jump
-> path_a {has_key}      # conditional GOTO
-> path_b {!has_key}     # like if/else branching

üí° GOTOs are automatic ‚Äî the player never sees them. Use them for branching logic based on game state!

Exiting Dialogue

Two ways to end dialogue ‚Äî choose based on your game flow:

# @end - Return to gameplay (exploration, walking around)
# Use this in RPGs where player can talk to NPCs again
[get_gear]
officer: "Your gear's in the locker."
@end  # Player can now walk around, open locker, talk again

# -> END - End the conversation completely
# Use for linear scenes or final farewells
[final_goodbye]
npc: "Farewell, hero. May we meet again."
-> END  # Conversation is over

üí° @end is for RPG exploration flow. -> END is for ending a scene. In most cases, prefer @end!

Conditions

Add {condition} to show choices only when conditions are met:

# Simple flag check
-> secret: "I know your secret" {found_diary}

# Negation (show if NOT set)
-> naive: "I trust you completely" {!knows_truth}

# Item check
-> use_key: "Use the key" {has_item:rusty_key}

# Companion check
-> ask_peng: "Peng, what do you think?" {companion:peng}

# Numeric comparison
-> brag: "I'm very experienced" {xp > 100}

# Complex conditions
-> special: "Special option" {has_key && talked_to_guard}

üí° Operators: && (and), || (or), ! (not), > < >= <= ==

Commands

Commands start with * and modify game state:

# Set a flag
*set talked_to_elder = true
*set player_name = Korra

# Add/subtract numbers
*add harmony = 10
*add xp = 50
*sub gold = 25

# Items
*give_item rusty_key
*remove_item old_map

# Companions
*add_companion peng
*remove_companion temporary_ally

# Custom commands (game-specific)
*start_combat bandit_encounter
*play_sound door_creak

üí° You can create custom commands like *start_combat ‚Äî your game decides how to handle them.

Game Events

Use @event to trigger dialogue from gameplay events (cutscenes, entering areas, etc.):

# Triggered when player enters the temple area
[temple_entrance]
@event:enter_temple
narrator: "The ancient temple looms before you..."
@end

# Triggered when exploration phase begins
[start_exploration]
@event:exploration_start
officer: "Scout the area. Report back when ready."
@end

üí° Use @talk:npc for NPC conversations, @event:name for scripted moments.

Stacked Nodes

Multiple choices can lead to the same content:

[choice_a]
[choice_b]
[choice_c]
npc: "Interesting choice..."
-> continue

All three nodes share the same dialogue ‚Äî perfect for converging branches.
